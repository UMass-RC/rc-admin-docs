    MGHPCC Annual Shutdown

    Note

    Items in italics should be done in-person

    Unity Cluster

    Power Down

    1. Block access from Palo Alto
    2. Shutdown all Compute Nodes from MAAS
    3. Shutdown the containers/VMs running on the head node in this order:
      1. Licensing LXC
      2. Web VM
      3. Login LXC
      4. Slurm VM
      5. NFS-GW VM
    4. Shutdown all NAS serving devices (excluding VAST)
    5. Shutdown VAST
      1. Deactivate the cluster. This can be done by clicking on the cluster in the VMS UI maintenance page and hitting pause/deactivate or using VCLI (cluster deactivate) to do the same. Wait until the entire cluster deactivates cleanly on its own.
      2. Connect to any cnode via ssh (web UI ip, username: vastdata, pass: vastdata), and run these commands: 
        #verify clush is healthy
clush -a hostname
#shutdown the cluster
clush -g cnodes 'sudo /usr/sbin/shutdown -h +2'; clush -g dnodes 'sudo /usr/sbin/shutdown -h +5'
      3. Disconnect power to the cnodes and dnodes, either in-person or through remote access to the PDU.
    6. Shutdown identity LXC on head node
    7. Shutdown head node

    Power Up

    1. Power up head node
    2. Power up identity container
    3. Power up VAST
      1. Plug in in the D-nodes and wait for them to power up. Wait 10 minutes to ensure the D-nodes start cleanly.
      2. Plug in the C-nodes and wait for them to power up. Wait 10 minutes to ensure the C-nodes start cleanly.
      3. From the web UI, activate the cluster.
    4. Power up head node
    5. Power up all NAS devices
    6. Power up head node VMs/containers in this order:
      1. NFS-gw VM
      2. Login Container
      3. Slurm VM
      4. Web VM
      5. Licensing Container
    7. Power up all compute nodes from MAAS
    8. Allow access from Palo Alto

    Perirhinal

    Power Down

    1. Pause all active VMs
    2. Shutdown from Web UI

    Power Up

    1. Power up in-person, IPMI is not available

    CloudLab / OCT

    Power Down

    1. Power down head node

    Power Up

    1. Power up head node

    Pikes

    Power Down

    1. From CMGUI, power down all compute nodes
    2. From CMGUI, power down head node, you will lose connection to Bright

    Power Up

    1. Power up head node in-person
    2. Power up compute nodes from CMGUI
