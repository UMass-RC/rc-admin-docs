Procedures for MGHPCC Shutdown

Unity Cluster

Power Down

  1. Block access from the Palo firewall rule.
  2. Shutdown all Compute Nodes
  3. Shutdown the containers/VMs running on the head node in this order:
  4. Licensing Container
  5. Web VM
  6. Slurm VM
  7. Login Container
  8. NFS-gw VM
  9. Identity Container
  10. Shutdown head node
  11. Shutdown NAS1
  12. Shutdown astro-th NAS
  13. Shutdown cee-water NAS
  14. Shutdown VAST
  15. Deactivate the cluster. This can be done by clicking on the cluster in the VMS UI maintenance page and hitting pause/deactivate or using VCLI (cluster deactivate) to do the same. Wait until the entire cluster deactivates cleanly on its own.
  16. Run these on any cnode to initiate shutdown. Cnodes shut down first, followed by Dnodes. #verify clush is healthy clush -a hostname #shutdown the cluster clush -g cnodes 'sudo /usr/sbin/shutdown -h +2'; clush -g dnodes 'sudo /usr/sbin/shutdown -h +5'
  17. Physically unplug power - the nodes come up automatically with power restoration

Power Up

  1. Power up VAST
  2. Plug in in the D-nodes and wait for them to power up. Wait 10 minutes to ensure the D-nodes start cleanly.
  3. Plug in the C-nodes and wait for them to power up. Wait 10 minutes to ensure the C-nodes start cleanly.
  4. From the web UI, activate the cluster.
  5. Power up cee-water NAS
  6. Power up astro-th NAS in person, IPMI not yet set up
  7. Poewr up NAS1
  8. Power up head node
  9. Power up head node VMs/containers
  10. Identity Container
  11. NFS-gw VM
  12. Login Container
  13. Slurm VM
  14. Web VM
  15. Licensing Container
  16. Power up all compute nodes
  17. Allow access from Palo Alto

Perirhinal

Power Down

  1. Pause all active VMs
  2. Shutdown from Web UI

Power Up

  1. Power up in-person, IPMI is not available

CloudLab / OCT

Power Down

  1. Power down head node

Power Up

  1. Power up head node

General

  1. After the above, shut down the PA-3220
